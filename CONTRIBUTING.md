# Como Contribuir

Bem-vindo ao **Wiki de Programação e Experiência de TI**! Este repositório tem como objetivo reunir conhecimentos sobre **programação, computação e experiências no setor de TI**, e está aberto para contribuições da comunidade.

Se você tem alguma experiência ou conhecimento para compartilhar, ou se encontrou algo que pode ser melhorado, sinta-se à vontade para contribuir!

## Importante: Respeito à Legislação e Licença

- **Este repositório é destinado apenas a conteúdos relacionados à programação, computação e experiências no setor de TI**. Por favor, **não contribua com materiais que violem direitos autorais, patentes ou qualquer outra legislação aplicável**.
  
- **Favor não violar leis**: As contribuições e o uso do repositório devem sempre estar em conformidade com as **leis do Brasil** e com a **Constituição Brasileira**. Qualquer violação legal será tratada de acordo com a legislação vigente.

- **Licença e Forks**: Se você deseja **fazer um fork** deste repositório, **por favor, entre em contato primeiro** para discutir sua intenção. Também **leia o arquivo de licença** antes de qualquer contribuição ou uso do repositório. O código está sob uma licença **Creative Commons - Atribuição-NãoComercial 4.0** e todas as contribuições devem seguir os termos dessa licença.

## Tipos de Contribuições Aceitas

Aqui estão algumas maneiras pelas quais você pode contribuir para o projeto:

- **Correções de erros**: Encontrou algum erro de digitação, código incorreto ou informação desatualizada? Nos avise!
- **Melhorias no conteúdo**: Se você acha que algum artigo pode ser melhorado ou expandido, sinta-se à vontade para editar.
- **Novos artigos e tutoriais**: Se você tem uma área específica de programação ou TI que gostaria de compartilhar, crie uma nova página ou artigo!
- **Sugestões**: Se você tiver ideias para novos tópicos ou melhorias no conteúdo, abra uma *issue* para discutir.
  
## Como Contribuir

1. **Faça um Fork** do repositório.
2. **Crie um novo branch** para sua contribuição. Nomeie o branch de forma clara, por exemplo: `adicionar-tutorial-python` ou `corrigir-erro-artigo`.
3. **Faça as modificações** no conteúdo. Se estiver adicionando novos artigos, use **Markdown** para manter o formato consistente com o resto do projeto.
4. **Teste suas alterações**: Verifique se tudo está funcionando como esperado. No caso de tutoriais ou códigos, certifique-se de que o código funciona corretamente.
5. **Envie um Pull Request**: Quando você estiver pronto, envie um PR com uma descrição clara das alterações feitas. Se for um **artigo novo**, adicione uma breve descrição sobre o que o artigo aborda.

## Regras de Contribuição

- **Atribuição**: O conteúdo deste repositório está sob a **Licença Creative Commons - Atribuição-NãoComercial 4.0 (CC BY-NC)**. Ao contribuir, você concorda em licenciar seu trabalho sob os mesmos termos. Atribua crédito adequado a qualquer fonte ou conteúdo que você esteja utilizando.
  
- **Qualidade do Conteúdo**: Ao enviar um artigo ou tutorial, tente manter a qualidade do conteúdo. Use uma linguagem clara, explicações concisas e código bem formatado.
  
- **Formato e Estilo**:  
  - Use **Markdown** para adicionar ou editar artigos.
  - Mantenha o formato de títulos, subtítulos e listas consistente com o restante do repositório.
  - Se for um tutorial ou código, **comente o código** e adicione explicações para facilitar o entendimento.
  
- **Evite conteúdo comercial**: Este projeto é dedicado a **compartilhar conhecimento técnico** de maneira aberta e não comercial. Por favor, evite qualquer conteúdo com intenção comercial ou promocional.

## Código de Conduta

Este projeto segue um **Código de Conduta** para garantir um ambiente saudável e colaborativo para todos os contribuintes. Por favor, seja respeitoso, educado e inclusivo.

## Como Relatar Problemas ou Sugerir Melhorias

Se você encontrar um problema ou tiver uma sugestão para melhorias, sinta-se à vontade para **abrir uma issue**. Se possível, forneça detalhes claros e exemplos que ajudem a entender a situação.
