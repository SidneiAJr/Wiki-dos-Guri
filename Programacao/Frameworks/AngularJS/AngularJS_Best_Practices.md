# AngularJS — Boas Práticas de Organização

A boa organização do projeto torna o código mais claro, fácil de manter e escalar.

---

## Estrutura Recomendada de Pastas
Separar o projeto em módulos organizados por função e não por tipo de arquivo.

---

## Principais Boas Práticas
- Manter um controlador para cada tela
- Usar services para lógica compartilhada
- Evitar colocar lógica no HTML
- Nomear arquivos de forma clara e padronizada
- Separar cada funcionalidade em um módulo próprio
- Documentar cada componente da aplicação

---

## Visão Geral
Seguir boas práticas mantém a aplicação limpa, compreensível e pronta para crescer de forma organizada.
